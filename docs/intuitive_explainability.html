The explainability analysis

Our workflow investigates the model's decision-making process by tracking the journey of input images across five crucial stages: prediction, explanation, intervention, error analysis and semantic interpretation.

At each step, we move progressively deeper into the model's black box, from the pixels that seem to matter, to the stability of that evidence, to the semantic concepts that may shape the final decision.

01

Spatial Attribution The where

Hypothesizes where the visual evidence driving the model's decision may be located within the image.

02

Controlled Testing The reliability

Empirically checks whether the model's focus is stable and faithful, or whether it depends on fragile visual shortcuts.

03

Conceptual Analysis The what

Translates activation patterns into human-readable attributes and tests whether the internal representation aligns with semantic concepts.

Methods at a glance

Two ways to make a prediction understandable

The report combines post-hoc explanation methods, which inspect a trained ResNet50 after it predicts, with an interpretable-by-design model, where concepts are built into the prediction path.

Post-hoc

Grad-CAM, Integrated Gradients and TCAV do not change the classifier. They ask what evidence the already-trained model appears to use, either in pixels, feature maps or concept directions.

Interpretable by design

The Concept Bottleneck Model changes the architecture: the class decision must pass through explicit semantic variables before the final prediction is made.

Spatial

Grad-CAM

Uses gradients on late convolutional maps to highlight broad image regions that support a class.

Pixel path

Integrated Gradients

Accumulates input gradients from a baseline image to estimate pixel-level contribution.

Concept probe

TCAV

Tests whether a class score is sensitive to directions representing human concepts in activation space.

Concept model

Concept Bottleneck

Predicts named concepts first, then predicts the class from those concepts.

Model and prediction

The explained model is a standard ResNet50.

We start from an ImageNet-pretrained ResNet50 and adapt it to the animal classes in our dataset. This transfer-learning setup preserves broadly useful visual features while specialising the model for our classification task.

Question 2: is the map stable and faithful?

A plausible map becomes useful only when its highlighted evidence survives interventions and controls the selected class score.

The analysis first changes the approximate background, then quantifies map stability and faithfulness, and finally applies the same logic to misclassified images.

Background interventions

Gaussian noise

Add random noise only outside the approximate animal mask.

Colour shift

Shift background RGB channels while leaving the animal region intact.

Background replacement

Replace the approximate background with controlled random texture.

Measured response

Quantified walrus-attractor hypothesis

Across the unique perturbed examples in Figure 06, 5 / 12 prediction transitions ended in walrus. Under background replacement alone, 3 / 4 examples ended in walrus. The repeated appearance of walrus under textured background interventions is strong enough to motivate a systematic scan.

Stability and faithfulness metrics

IoU and Spearman measure whether the explanation remains spatially consistent. Deletion and insertion test whether highly ranked pixels actually control the target score. These checks answer different parts of the reliability question and must be interpreted together.

Error analysis

Misclassifications are useful because they force the explanation to account for a decision that is visibly incorrect.

An attribution for the predicted class asks which evidence increased the score the network actually chose. An attribution for the true class asks a different question: which regions would need to support the correct class more strongly?

Misclassification reading

Wrong target versus true target

In Figures 15a and 15b, each row shows the same image explained twice. The wrong-target map explains the class actually predicted by the model. The true-target map explains the ground-truth class.

Pattern 1

Wrong map on background, true map on animal

The predicted class is supported by contextual evidence. This is the clearest shortcut pattern: the model finds support for the wrong class outside the animal.

Pattern 2

Wrong map on animal, true map on animal

Both classes compete on similar object evidence. The error is more consistent with morphology, pose, texture or visually similar body structure.

Pattern 3

Wrong map on animal, true map on background

The model sees the animal, but the animal features activate the wrong class more strongly than the true class. The heatmap can look plausible while supporting a false decision.

A saliency map can be faithful to a false prediction.

A heatmap centered on the animal is not automatically a correct explanation. It may faithfully describe what increases the wrong class score. This separates visual plausibility from faithfulness and from ground-truth alignment.

Interpretability limit exposed by errors

The failure is not only that saliency maps may move to the background. A deeper failure is that they may faithfully explain a false model decision. A wrong-target map can be faithful to the model score and still be semantically misleading because it assigns animal evidence to the wrong class.

For this reason, visual inspection is insufficient.

Question 3: which semantic concepts are represented?

AwA2 attributes let the analysis move beyond unnamed pixels and ask whether the ResNet50 representation contains human-readable semantic structure.

Concept sensitivity is not concept truth

TCAV asks whether a class score is sensitive to a learned concept direction. It does not prove that the animal visibly contains that concept, and it does not by itself measure dataset correlation. When the AwA2 oracle value is low but TCAV remains high, the pair is best read as an audit candidate for an entangled direction: the CAV may contain anatomy, texture, background, habitat and co-occurring class evidence at the same time.

Image-level rerun

The bridge audit is still class-level. To test whether the suspicious directions actually behave on individual images, the model was rerun on 24 target images per pair. For each image the audit computes the directional derivative along the CAV before and after background perturbations. This separates a stable concept effect from a context-sensitive response.

Main image-level conclusion

The suspicious pairs do not all mean the same thing. flippers → polar bear is not a strong hidden flipper concept: the positive-direction rate is only 50.0% and falls to 16.7% after background replacement. In contrast, horns → dolphin and hooves → dolphin remain directionally positive for many images, but the dolphin prediction itself is highly unstable under the same intervention. The result is therefore not "these animals have the wrong attributes"; it is "some TCAV directions are entangled with class/context evidence".

Question 4: can prediction be mediated by concepts?

The bottleneck experiment asks whether prediction can be mediated by named attributes instead of explained only after the fact.

Class accuracy

The current CBM reaches approximately 0.799, compared with 0.857 for the ResNet50 baseline.

Concept accuracy

Binary concept accuracy is approximately 0.754. Concept quality varies substantially across attributes.

Intervention

Changing a predicted concept tests how that semantic variable affects the CBM class head.

Model boundary

CBM concept contributions explain the CBM pathway: ResNet50 backbone, concept head and class head. They are a semantic comparator, not an explanation of the separate end-to-end ResNet50 baseline used for local attribution maps.

What is an oracle intervention?

An oracle intervention is a controlled edit of the bottleneck, not an edit of the image.

For an image with true class y, the CBM first predicts a concept vector ĉ(x). AwA2 supplies a class-level reference profile c*(y). A one-concept oracle intervention replaces one predicted coordinate ĉj(x) with the corresponding reference value c*j(y), keeps every other predicted concept fixed, and runs the edited vector through the same class head.

Why “oracle”?

The evaluator uses privileged knowledge of the true class and its AwA2 attribute profile. The model does not receive this information during normal inference.

What is measured?

The audit measures changes in the true-class probability, the predicted-versus-true logit margin and the final top-1 class after the semantic coordinate is corrected.

What it does not prove

It is not a pixel-level causal edit and not an image-specific ground-truth annotation. AwA2 targets are class-level, and edited vectors can lie outside the distribution seen by the class head during training.

Single versus full oracle

A single intervention changes one concept and isolates its local effect. A full intervention replaces the complete predicted vector and stress-tests whether the class head can recognize the true-class AwA2 profile.

Probability and rank can move differently

In a multiclass softmax, a correction can make the true class top-1 even if its absolute probability decreases, provided that the competing classes decrease by more. For this reason the report reads Δp(true), the wrong-versus-true margin and the final predicted class together.

CBM real error decomposition: bobcat → hippopotamus

This case is selected directly from the CBM error report with cbm_correct = False, ranked by mean absolute concept error.

The true class is bobcat, while the Concept Bottleneck Model predicts hippopotamus. The contrast class is therefore the CBM's own wrong top-1 prediction, not a manually fixed comparator.

CBM real error decomposition: polar bear → sheep

This is the second strongest real CBM error under the same ranking rule.

The true class is polar+bear, while the Concept Bottleneck Model predicts sheep. Here the oracle class-level concept vector would recover polar+bear, so the error is consistent with a bottleneck concept-prediction failure.

Class-level supervision

AwA2 attributes are class-level rather than image-specific. Every image from one class receives the same target concept vector, even when an attribute is occluded or not visible.

Limitations and future work

The report is intentionally conservative: every result is interpreted only at the level supported by the experiment that produced it.

Conclusion

Question Answer supported by the project
1. Where is evidence located? Grad-CAM and IG provide different spatial hypotheses for a selected class.
2. Is the map stable and faithful? Only partially. Background interventions and deletion expose instability, shared evidence and contextual sensitivity.
3. Which concepts are represented? Validated TCAV identifies measurable semantic sensitivity, subject to concept coverage and random controls.
4. Can concepts mediate prediction? The CBM creates an explicit concept path, with lower class accuracy and imperfect concept prediction.

The strongest defensible conclusion remains conservative: saliency maps are useful diagnostic tools, but they should not be presented alone as faithful explanations of a deep visual classifier.
