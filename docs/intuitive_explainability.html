Explainability Audit for Visual Classifiers

A structured study of whether local saliency maps and concept-level explanations provide reliable evidence for a ResNet50 animal classifier trained on AwA2.

What explainability means here

Explainability connects a model output to interpretable evidence. Here, the output is an animal class probability and the evidence can be either image regions or human-readable concepts.

ImageRGB pixels are resized, cropped and normalized.
ClassifierResNet50 produces logits and softmax probabilities.
Local mapsGrad-CAM and Integrated Gradients produce heatmaps.
Stress testsBackground changes test map stability.
ConceptsTCAV and CBM test semantic evidence.

Saliency map

A saliency map assigns an importance value to spatial locations. It is usually displayed as a heatmap.

Local explanation

A local explanation is computed for one image and one target class.

Faithfulness

A faithful explanation should identify evidence that actually affects the target score.

Concept explanation

A concept explanation uses semantic attributes such as stripes, horns, paws or color.

Classifier and training

The explained model is a ResNet50 image classifier. Most architectural choices are standard defaults from torchvision.models.resnet50: the convolutional stem, residual bottleneck blocks, kernel sizes, skip connections and downsampling pattern are not manually redefined in this project. The project-specific choices are the input preprocessing, the replacement of the final classifier, the fine-tuning policy and the training objective.

Default backbone

The backbone is the canonical ResNet50 architecture. Unless --no-pretrained is passed, it starts from ImageNet pretrained weights. This is a transfer-learning setup: early visual filters already detect generic patterns such as edges, textures and object parts.

Input tensor

Each RGB image is resized/cropped to 224 x 224, converted to a tensor with shape 3 x 224 x 224 and normalized with ImageNet mean/std. The model therefore receives normalized channels, not raw pixel values.

First convolution

ResNet50 starts with a 7 x 7 convolution with stride 2, followed by batch normalization, ReLU and max pooling. This quickly reduces spatial resolution while increasing feature abstraction.

Residual bottlenecks

Each bottleneck block uses 1 x 1, 3 x 3 and 1 x 1 convolutions. The residual skip connection adds the block input to the transformed representation, making optimization easier in a deep network.

Downsampling and Grad-CAM layer

Spatial resolution is reduced by the initial stride and by strided residual stages. The last convolutional representation is approximately 2048 x 7 x 7. Grad-CAM is computed at layer4[-1], so its native heatmap is coarse and must be upsampled for visualization.

Classifier head

The original ImageNet fully connected layer is replaced by a new linear layer with one output per AwA2 class in the manifest. The network outputs logits z. Softmax is used after the logits to obtain class probabilities for interpretation.

Fine-tuning policy

The current baseline fine-tunes layer4 and fc. Earlier layers are frozen to preserve generic ImageNet features and reduce training cost. Frozen BatchNorm layers are forced to evaluation mode so their running statistics do not change during training.

Regularization

There is no additional dropout layer in the standard ResNet50 classifier used here. Regularization comes from transfer learning, frozen early layers, data augmentation on the training split, AdamW weight decay and validation-based checkpoint selection.

Loss function

The baseline is optimized with PyTorch nn.CrossEntropyLoss(), which combines log-softmax and negative log-likelihood for multiclass classification.

L(x, y) = - log( exp(z_y) / sum_j exp(z_j) )

The loss penalizes the model when the logit assigned to the true animal class is not sufficiently larger than the logits assigned to the other classes.

Optimizer

The script uses torch.optim.AdamW over trainable parameters only. The default command-line values are lr = 1e-4 and weight_decay = 1e-4. Weight decay discourages unnecessarily large weights in the fine-tuned layers.

Training schedule

The default script runs for 5 epochs with batch size 32. During training, the project records train/validation loss and accuracy. The saved checkpoint is the model with the best validation accuracy observed during the run.

Preprocessing policy

Training uses stochastic augmentation: random resized crop and horizontal flip. Validation and test use deterministic resize plus center crop. This separation avoids measuring validation performance on randomly augmented evaluation images.

Local attribution methods

Grad-CAM

Grad-CAM explains a target logit through the last convolutional feature maps.

alpha_k^c = mean( dS_c / dA_k )
M_c = ReLU( sum_k alpha_k^c A_k )

The heatmap is semantically rich but spatially coarse.

Integrated Gradients

Integrated Gradients attributes the score difference between a blurred baseline and the input.

IG_i(x) = (x_i - x'_i) integral_0^1 dF(x' + a(x-x')) / dx_i da

The blurred baseline preserves global brightness while removing detailed morphology.

When the network is wrong

A misclassification is the most informative case for explainability. The model has produced a high-dimensional representation, assigned logits to all classes, and selected the class with the largest softmax probability. If the selected class is not the ground truth, the explanation should clarify what evidence supported the wrong decision.

Predicted-class explanation

For an incorrect prediction, Grad-CAM and Integrated Gradients are usually computed with respect to the predicted class. This answers the question: which regions increased the score of the class the model actually chose?

True-class explanation

The same methods can also target the true class. This answers a different question: which regions would need to support the correct class more strongly? Comparing predicted-class and true-class maps is useful when two animals share morphology.

Confidence is not correctness

A wrong prediction can still have high confidence. Softmax only normalizes logits; it does not verify whether the model used the right visual evidence. A confident error is therefore a strong candidate for shortcut or class-prototype analysis.

What the heatmap can reveal

If saliency is on the animal, the error may come from morphological similarity. If saliency is on the background, the model may be using context. If saliency is diffuse, the image may not contain a clean visual signal for the selected class.

Grad-CAM on an error

Grad-CAM localizes late convolutional evidence for the wrong class. If the highlighted region is the head, body shape or texture of the animal, the error is likely related to class similarity. If the highlighted region is scenery, the prediction may be relying on contextual correlation.

Integrated Gradients on an error

Integrated Gradients shows which input pixels accumulated attribution from the blurred baseline to the real image. On errors, scattered or background-heavy attribution warns that the wrong class score may be sensitive to low-level texture rather than robust morphology.

Conservative interpretation: an error heatmap does not prove why the model failed. It provides a hypothesis. The hypothesis becomes stronger only if it is consistent with perturbation stability, class-specific attribution and concept-level evidence.

Background stress test

The central question is whether explanations remain stable when the animal region is preserved but the background is changed.

Gaussian noise

Add random noise to the approximate background.

Color shift

Shift RGB channels only in the approximate background.

Background replacement

Replace the approximate background with uniform random texture.

Interpretation

If the predicted class or heatmap moves substantially, the explanation is sensitive to context.

Concept-level explanations

Concept-level methods complement heatmaps by asking whether the representation aligns with human-readable attributes.

TCAV

TCAV learns a concept direction in activation space from positive concept images and negative references.

TCAV(c,k,l) = mean_x 1[ grad h_l,k(x) dot v_c^l > 0 ]

A high score means moving in the concept direction tends to increase the target class score.

Concept Bottleneck Model

A CBM changes the prediction path: image to predicted concepts, then concepts to class.

L = CE(y, y_hat) + lambda BCE(c, c_hat)
p(y|x) = softmax(g(c_hat))

Concepts become an explicit interface for inspecting and intervening on predictions.

Final interpretation

The project should be presented as an audit pipeline. Grad-CAM and Integrated Gradients provide local hypotheses about image evidence. Background stress tests check whether those hypotheses remain stable. TCAV and the Concept Bottleneck Model add semantic structure.

A heatmap is not proof. It becomes more credible only when visual inspection, perturbation stability, faithfulness curves, class specificity and concept evidence agree.

The strongest conclusion is conservative: post-hoc saliency maps are useful diagnostic tools, but they should not be presented alone as faithful explanations of a deep visual classifier.
