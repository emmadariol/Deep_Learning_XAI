How do we make a prediction understandable?

  1. 01 PREDICTION

    We start from a predicted animal class and ask what supports it.

  2. 02 EVIDENCE

    Inspection of the image evidence: attribution maps to see which regions/pixels support that class score.

  3. 03 CHECKS

    Test whether that evidence is reliable: change the input/remove highlighted regions to test stability and faithfulness.

  4. 04 CONCEPTS

    Check whether the model can be read through human concepts: can attributes such as stripes, horns or flippers shape the decision?

Methods

Post-hoc methods: analyse a trained net after a prediction has already been produced.
Interpretable by design: it changes the prediction path itself, forcing the class decision through named attributes.

After the prediction post-hoc
  • Grad-CAM → locates class evidence in late feature maps.
  • Integrated Gradients → attributes the class score to input pixels.
  • TCAV → tests sensitivity to semantic concept directions.
  • Stress tests → evaluate stability and faithfulness.
In the prediction path interpretable by design
  • Concept Bottleneck Model (CBM) → makes concepts part of the prediction path.
  • Concept head → predicts 20 semantic attributes.
  • Class head → predicts the animal only from concepts.
  • Interventions → test the effect of concept corrections.

Model and prediction

Before explaining a decision, we first define the model, the signal it produces and the internal representations available for inspection.

The baseline is an ImageNet-pretrained ResNet50 adapted to the 20 animal classes in the maintained AwA2 subset. It maps each normalized image to one score per class, providing the common prediction target for all post-hoc analyses in this report.

From pixels to class probabilities

ResNet50 progressively compresses spatial information while building higher-level features. Global average pooling converts the final feature maps into a 2048-dimensional vector, and the linear head produces 20 raw class scores, or logits. Softmax then converts those scores into probabilities; the largest probability determines the top-1 prediction.

How the classifier learns

During fine-tuning, cross-entropy compares the logits with the ground-truth class. The loss encourages the correct-class logit to exceed the competing scores, while AdamW updates only layer4 and the final fc layer.

Objective

Cross-entropy training signal

CrossEntropyLoss compares class logits with the ground-truth label. AdamW then updates the trainable parameters with weight decay.

Why the internal representation matters

The forward trace below follows a correctly classified antelope through the network. Early activations retain edges, texture and fine spatial detail; deeper stages become smaller and more selective. By layer4, the representation has collapsed to a coarse 7 × 7 grid that responds strongly to the animal region.

This progression determines what each explanation method can observe: Integrated Gradients attributes the input pixels, TCAV probes the layer3 representation, and Grad-CAM localizes evidence in the final convolutional stage.

What the explanations target

Grad-CAM, Integrated Gradients and TCAV differentiate a selected raw class logit, before softmax. Probabilities summarize confidence and determine the top-1 class, but the attribution target is the class score itself.

Question 2: is the map stable and faithful?

A plausible map becomes useful only when its highlighted evidence survives interventions and controls the selected class score.

The analysis first changes the approximate background, then quantifies map stability and faithfulness, and finally applies the same logic to misclassified images.

Reliability checks

Stability and faithfulness metrics

These measures are read together because they answer two different questions: whether the explanation stays consistent under controlled changes, and whether the highlighted evidence actually affects the target score.

Stability

Does the map stay similar when the input changes?

It checks whether attribution remains in the same regions, or at least keeps a similar importance ranking, after controlled background interventions.

Faithfulness

Do the highlighted pixels actually matter for the score?

It checks whether removing or inserting salient evidence changes the target class score in the expected direction.

Stability

Top-k IoU

Asks whether the most salient spatial support remains in the same region. Higher overlap means more stable localization.

Stability

Spearman rank

Asks whether the full pixel-importance ordering remains similar. Constant maps are treated as undefined rather than artificial agreement.

Faithfulness

Deletion and insertion

Asks whether salient pixels control the target score. A faithful map should lose evidence quickly and recover it early during insertion.

Faithfulness

Region allocation

Asks how much normalized saliency lies on the approximate animal region compared with the background.

Error analysis

Misclassifications are useful because the model made a visibly wrong decision: the explanation must therefore show what pushed the network toward that wrong class, not just where the heatmap looks plausible.

Misclassification reading

Wrong target versus true target

In Figures 15a and 15b, each row shows the same image explained twice. The wrong-target map explains the class actually predicted by the model. The true-target map explains the ground-truth class.

Pattern 1

Wrong map on background, true map on animal

The predicted class is supported by contextual evidence. This is the clearest shortcut pattern: the model finds support for the wrong class outside the animal.

Pattern 2

Wrong map on animal, true map on animal

Both classes compete on similar object evidence. The error is more consistent with morphology, pose, texture or visually similar body structure.

Pattern 3

Wrong map on animal, true map on background

The model sees the animal, but the animal features activate the wrong class more strongly than the true class. The heatmap can look plausible while supporting a false decision.

A saliency map can be faithful to a false prediction.

A heatmap centered on the animal is not automatically a correct explanation. It may faithfully describe what increases the wrong class score. This separates visual plausibility from faithfulness and from ground-truth alignment.

Interpretability limit exposed by errors

The failure is not only that saliency maps may move to the background. A deeper failure is that they may faithfully explain a false model decision. A wrong-target map can be faithful to the model score and still be semantically misleading because it assigns animal evidence to the wrong class.

For this reason, visual inspection is insufficient.

We can use the AwA2 attributes to move from pixel-level explanations to semantic concepts and test whether ResNet50 encodes recognizable animal traits.

Model concept hallucinations

These mismatches do not mean that the model literally sees nonexistent attributes; they indicate that a learned concept direction may also encode shape, texture, habitat or background cues.

Do concept hallucinations persist under background perturbations?

To test whether the suspicious directions actually behave on individual images, the model was rerun on 24 target images per pair. For each image the audit computes the directional derivative along the CAV before and after background perturbations. This separates a stable concept effect from a context-sensitive response.

Question 4: can prediction be mediated by concepts?

CBM (Concept Bottleneck Model)

The bottleneck experiment asks whether prediction can be mediated by named attributes instead of explained only after the fact.

CBM training configuration Backbone initialization, semantic bottleneck and optimization used for the concept-mediated model.

The CBM backbone starts from the AwA2 baseline checkpoint and fine-tunes layer4. Its bottleneck contains 20 selected AwA2 concepts with dropout 0.15. Training uses batch size 16, AdamW with learning rate and weight decay 1e-4, five epochs, and equal weights of 1.0 for the concept and class losses.

Conclusion

  • The project shows that heatmaps, perturbations, faithfulness metrics, error analysis and concept-based methods must be interpreted together. Pixel-wise and concept-based methods answer complementary questions: Grad-CAM and Integrated Gradients show where evidence appears, while TCAV and the CBM help inspect which semantic information may be involved.
  • Background stress tests reveal that the model can rely on contextual evidence rather than animal morphology alone. This affects both predictions and explanations, so saliency maps should be treated as hypotheses to validate.
  • Faithfulness checks and error analysis are essential: a heatmap can look plausible and still explain a wrong decision.
  • Every method has assumptions: Grad-CAM is coarse, Integrated Gradients is baseline-sensitive, TCAV can capture correlated latent patterns, and the CBM depends on concept quality.
  • The analysis is further limited by approximate background masks, class-level AwA2 concepts and the use of a controlled subset of the dataset.
  • Interpretability remains partial: explanations highlight relevant associations but do not guarantee a causal understanding of model behavior.
  • In high-stakes domains such as medical imaging, this workflow could support model auditing by checking not only whether a prediction is correct, but whether it is supported by clinically meaningful evidence. In radiology, explanations should highlight lesions, tissue abnormalities or relevant anatomical regions rather than scanner artifacts, text markers, acquisition differences or hospital-specific shortcuts. Perturbation tests, faithfulness metrics and concept-level analysis can help distinguish predictions that are correct for valid clinical reasons from predictions driven by spurious dataset biases.
