Auditing explanations for a visual classifier

A heatmap can look persuasive without being reliable. This report tests whether local saliency maps and concept-level explanations provide evidence that survives controlled checks.

The explainability analysis

The project follows one image from prediction to explanation, intervention, error analysis and semantic interpretation.

Each stage answers a stronger question. Spatial attribution proposes where evidence may be located; controlled tests determine whether that proposal is stable and faithful; concept methods then examine whether the representation can be related to human-readable attributes.

Question 1

Where is evidence located? Grad-CAM and Integrated Gradients produce spatial maps for a selected class.

Question 2

Is the map stable and faithful? Perturbations and quantitative metrics test whether the highlighted evidence survives controlled checks.

Question 3

Which semantic concepts are represented? AwA2 attributes and TCAV connect activation directions to concepts such as stripes, horns, fur and flippers.

Question 4

Can prediction be mediated by concepts? A Concept Bottleneck Model forces the class prediction through an explicit semantic vector.

Working principle

These questions are complementary. A local heatmap cannot replace concept analysis, and concept sensitivity cannot replace a spatial explanation.

Model and prediction

The explained model is a standard ResNet50 adapted to the animal classes available in the AwA2 manifest.

We normally start from an ImageNet-pretrained ResNet50 and adapt it to the animal classes in our dataset. This transfer-learning setup preserves broadly useful visual features while specialising the model for our classification task. We can also rerun the training without ImageNet weights when we want a controlled comparison with an unpretrained initialization.

Input tensor

RGB images are resized and cropped to 224 × 224, converted to a 3 × 224 × 224 tensor and normalized with ImageNet statistics.

Residual backbone

The canonical bottleneck architecture uses skip connections and progressive downsampling. The last convolutional representation is approximately 2048 × 7 × 7.

Objective

CrossEntropyLoss compares class logits with the ground-truth label. AdamW updates trainable parameters with weight decay.

L = − log ( exp ( z y ) ∑ j exp ( z j ) )

Evaluation policy

Training uses stochastic augmentation. Validation and test images use deterministic resize and centre crop so reported performance is reproducible.

Question 1: where is evidence located?

Grad-CAM and Integrated Gradients locate candidate evidence for one image and one selected class.

A saliency map is the numerical attribution tensor. A heatmap is its colour-coded visualization over the image. Both methods produce maps, but they define importance differently and should not be treated as interchangeable measurements.

Scope of Question 1

A spatial map answers where attribution is concentrated. Stability, necessity and sufficiency are evaluated later through perturbation, deletion and insertion tests.

Question 2: is the map stable and faithful?

A plausible map becomes useful only when its highlighted evidence survives interventions and controls the selected class score.

The analysis first changes the approximate background, then quantifies map stability and faithfulness, and finally applies the same logic to misclassified images.

Background interventions

Gaussian noise

Add random noise only outside the approximate animal mask.

Colour shift

Shift background RGB channels while leaving the animal region intact.

Background replacement

Replace the approximate background with controlled random texture.

Measured response

Track prediction transitions, fixed-target confidence and spatial similarity between saliency maps.

Why the error maps are separate. Figures 07 and 08 are stability checks: they ask whether a fixed target explanation changes when only the background is perturbed. A misclassification is a different diagnostic. When the network is wrong, the saliency map should be computed for the class actually selected by the model and then compared with the saliency map for the ground-truth class. That target-specific error audit is shown in the Error analysis section.

Stability and faithfulness metrics

IoU and Spearman measure whether the explanation remains spatially consistent. Deletion and insertion test whether highly ranked pixels actually control the target score. These checks answer different parts of the reliability question and must be interpreted together.

Top-k IoU

Does the most salient spatial support remain in the same region? Higher overlap indicates more stable localization.

Spearman rank

Does the full pixel-importance ordering remain similar? Constant maps are treated as undefined rather than as artificial agreement.

Deletion and insertion

Do salient pixels actually affect the target score? A faithful map should delete evidence quickly and recover it early during insertion.

Region allocation

How much normalized saliency lies on the approximate animal region compared with the background?

Sanity controls

Does the map agree with occlusion evidence and change when relevant model parameters are randomized?

Error analysis

Misclassifications are useful because they force the explanation to account for a decision that is visibly incorrect.

An attribution for the predicted class asks which evidence increased the score the network actually chose. An attribution for the true class asks a different question: which regions would need to support the correct class more strongly?

Confident errors

Softmax normalizes logits; it does not verify the evidence. A wrong prediction can therefore remain highly confident.

Morphology or context

Animal-centred saliency suggests visual similarity between classes. Background-heavy saliency suggests possible contextual shortcuts.

Fixed target

Confidence changes under perturbation are measured for the original saliency target, even when the predicted label changes.

Cross-method evidence

Agreement between methods raises the priority of a case. Independent perturbation and sanity checks then determine whether the shared pattern controls the score.

Contrastive error audit

The original error grid targets only the class selected by the network. The contrastive audit fixes two targets on the same image: the original wrong prediction and the ground-truth class. It then measures whether controlled interventions weaken the wrong class relative to the true class.

m ( x ) = z wrong ( x ) − z true ( x )

Target contrast

Grad-CAM and Integrated Gradients are computed twice on each error: once for the fixed wrong class and once for the true class. Similar maps for different targets indicate weak class discrimination.

Decision margin

The original error has m(x) > 0. A positive margin reduction after perturbation means the intervention made the true class more competitive.

Deletion faithfulness

Pixels ranked by each target map are progressively replaced with a blurred baseline. A faithful wrong-target map should produce a substantial fall in the wrong-class score.

Semantic comparator

The CBM decomposes its own linear wrong-versus-true margin into concept contributions. This explains the CBM pathway and is not presented as causal evidence for the separate ResNet.

Current audit snapshot: two selected errors

All three background interventions changed the top-1 prediction in both examples, but none corrected the image to its true class. Background replacement produced the largest mean reduction of the fixed wrong-versus-true margin (0.638). This supports contextual sensitivity, but not successful recovery of the correct decision.

For wolf → antelope, replacement reduced the margin from +0.691 to +0.119 and the model moved to walrus. For polar bear → tiger, the margin crossed zero from +0.509 to −0.194, so polar bear outranked tiger, yet walrus remained the global top-1 class. Removing the top 20% wrong-target pixels reduced the margin by about 0.36 on average for both Grad-CAM and Integrated Gradients.

The original wrong-versus-true map overlap was lower for Grad-CAM (IoU 0.203) than for Integrated Gradients (IoU 0.420), while the corresponding mean rank correlations were 0.108 and 0.733. In these cases, Integrated Gradients assigned more similar pixel rankings to competing targets. The sample is deliberately small and serves as an error-level case study, not a test-set estimate.

Decision rule

A background-supported error requires aligned evidence: the intervention should reduce the fixed wrong score or margin, the wrong-target map should move, and deletion should confirm that the highlighted region controls the score. A heatmap alone does not establish this conclusion.

Question 3: which semantic concepts are represented?

AwA2 attributes let the analysis move beyond unnamed pixels and ask whether the ResNet50 representation contains human-readable semantic structure.

This section reports the concept-level audit and its validated outputs. The methodological details are kept with the clickable TCAV figure.

Validated TCAV protocol

The current run uses 20 repeated CAV fits per evaluated pair, held-out validation, matched random directions, confidence intervals, permutation tests and Benjamini–Hochberg correction. Seven of 49 evaluated pairs remain significant at corrected alpha 0.05. Unexpected pairs are kept visible so they can be audited in the heatmap and effect-size views.

Interpretation boundary

Read the TCAV outputs as representation-level evidence. They do not mark pixels in an individual image and do not establish a causal mechanism by themselves.

Question 4: can prediction be mediated by concepts?

The bottleneck experiment asks whether prediction can be mediated by named attributes instead of explained only after the fact.

This section reports the CBM results and interventions. The architectural details are kept with the clickable CBM figures.

Class accuracy

The current CBM reaches approximately 0.799, compared with 0.857 for the ResNet50 baseline.

Concept accuracy

Binary concept accuracy is approximately 0.754. Concept quality varies substantially across attributes.

Intervention

Changing a predicted concept tests how that semantic variable affects the CBM class head.

Model boundary

CBM concept contributions explain the CBM pathway: ResNet50 backbone, concept head and class head. They are a semantic comparator, not an explanation of the separate end-to-end ResNet50 baseline used for local attribution maps.

Class-level supervision

AwA2 attributes are class-level rather than image-specific. Every image from one class receives the same target concept vector, even when an attribute is occluded or not visible.

Conclusion

The project becomes coherent when every method answers one explicit question and no result is asked to support a stronger claim than it can justify.

Question Answer supported by the project
1. Where is evidence located? Grad-CAM and IG provide different spatial hypotheses for a selected class.
2. Is the map stable and faithful? Only partially. Background interventions and deletion expose instability, shared evidence and contextual sensitivity.
3. Which concepts are represented? Validated TCAV identifies measurable semantic sensitivity, subject to concept coverage and random controls.
4. Can concepts mediate prediction? The CBM creates an explicit concept path, with lower class accuracy and imperfect concept prediction.

The final interpretation combines prediction behaviour, spatial attribution, controlled interventions, quantitative metrics and concept evidence. Agreement strengthens a hypothesis; disagreement identifies a limitation that should remain visible.

The strongest defensible conclusion remains conservative: saliency maps are useful diagnostic tools, but they should not be presented alone as faithful explanations of a deep visual classifier.

Final caution

A heatmap is not proof. It becomes more credible only when visual inspection, perturbation stability, faithfulness, class specificity and concept evidence tell a consistent story.
