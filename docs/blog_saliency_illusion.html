The Illusion of Saliency Maps

A ResNet50 can classify an animal correctly and still rely on fragile visual shortcuts. This post walks through Grad-CAM, Integrated Gradients, and a background stress test to show why heatmaps are useful, but not enough.

50 AwA2 animal classes available in the full dataset.
3 saliency methods used: input gradients, Grad-CAM, Integrated Gradients.
3 background perturbations: noise, color shift, random replacement.

The Problem

Saliency maps are seductive. They turn a neural network into an image we can look at: red regions mean important, blue regions mean less important. The danger is that a visually plausible heatmap can feel like an explanation even when it is only a fragile trace of gradients and correlations.

Core question: when the model predicts an animal, is the decision driven by morphology, or by shortcuts such as background, color, texture, and recurring dataset context?

What would a good explanation show?

If the model predicts zebra, a convincing explanation should emphasize stripes, body shape, head, legs, and other animal-specific features. If it mostly lights up grass, sky, water, or image borders, then the explanation is telling a more complicated story.

The Experimental Pipeline

The project uses Animals with Attributes 2. A ResNet50 baseline is trained or loaded from a checkpoint, then several explanation methods are applied to both correct and incorrect predictions.

Phase 1 Prepare AwA2 manifests and DataLoaders.
Phase 2 Train or load a ResNet50 classifier.
Phase 3 Generate saliency maps for correct and wrong predictions.
Phase 4 Perturb approximate background regions.
Phase 5 Measure saliency degradation quantitatively.

Mathematical Setup

Let an RGB image be a tensor x in R^(3 x H x W). The classifier is a neural network f_theta that maps the image to one logit per animal class. Explanations are computed with respect to one target class c, usually the predicted class or the ground-truth class when we want to inspect a known failure. 

z = f_theta(x) in R^C
p(y = c | x) = softmax(z)_c = exp(z_c) / sum_j exp(z_j)
y_hat(x) = argmax_c p(y = c | x)
Prediction problem The model is optimized to make y_hat(x) match the label. Training minimizes cross-entropy, so the learned representation is rewarded for predictive evidence, even when that evidence is a contextual shortcut.
Explanation problem A saliency method returns a map S_c(x) in R^(H x W). Large values indicate pixels or regions that the method considers influential for class c. This is an attribution statement, not a proof of causality. 
CrossEntropy(z, y) = -log p(y | x)

Saliency objective:
S_c(x) should identify image regions that support score z_c(x).
The central scientific risk is confounding: if background and animal class are correlated in the dataset, gradients may highlight background because it is predictive, not because it is semantically part of the animal.

Algorithm 1: Input Gradients

Input gradients are the most direct saliency method. We ask how sensitive the class score is to each input pixel. If changing a pixel slightly would change the class score a lot, that pixel receives high saliency. 

G_c(x) = partial z_c(x) / partial x

For each pixel location (i, j):
S_c(i, j) = sum_{k in {R,G,B}} | G_c(k, i, j) |

The tensor G_c(x) has the same shape as the input image. It measures local sensitivity: if a very small perturbation delta x is applied, the first-order approximation of the class logit is 

z_c(x + delta x) approx z_c(x) +
This is easy to compute and useful for debugging, but it is noisy. High-frequency artifacts can dominate the map, making the explanation hard to interpret semantically.

Algorithm 2: Grad-CAM

Grad-CAM does not work directly at the pixel level. It looks inside the last convolutional block of the network, where spatial feature maps still exist but are more semantic than raw pixels.

1. Forward pass

The image is passed through ResNet50 and the target class score is selected.

2. Gradient weights

We compute gradients of the class score with respect to feature maps in layer4[-1].

3. Heatmap

Feature maps are weighted by those gradients, summed, ReLU-filtered, and resized to the image.

A in R^(K x U x V)              feature maps from ResNet50 layer4[-1]
A^k                              kth feature map
Z = U * V                         number of spatial positions

alpha_k^c = (1 / Z) * sum_i sum_j partial z_c / partial A^k_ij

L_GradCAM^c(i, j) = ReLU( sum_k alpha_k^c * A^k_ij )
S_c(x) = resize(L_GradCAM^c, H, W)

The coefficient alpha_k^c estimates how useful feature map k is for increasing class score z_c. The final ReLU keeps only positive evidence: regions that increase the score for the selected class are retained, while negative evidence is suppressed.

Grad-CAM is usually smoother and more readable than input gradients. But it is still post-hoc: it visualizes class-sensitive regions, not necessarily causal evidence.

Algorithm 3: Integrated Gradients

Integrated Gradients asks a more careful question: how does the class score change as the image moves from a baseline to the real input? 

x' = baseline image
x(alpha) = x' + alpha * (x - x'),      alpha in [0, 1]

IG_i^c(x) = (x_i - x'_i) * integral_0^1
            partial z_c(x' + alpha * (x - x')) / partial x_i d alpha

In practice, the integral is approximated with m discrete steps. This is why Integrated Gradients is much slower than Grad-CAM: every image requires many forward and backward passes along the path from baseline to input. 

IG_i^c(x) approx (x_i - x'_i) * (1 / m) * sum_{r=1}^m
                 partial z_c(x' + (r / m) * (x - x')) / partial x_i

Pixel map:
S_c(i, j) = sum_channels | IG^c(k, i, j) |

In this project, the baseline is not a black image. A black image is artificial and can create strange attributions. Instead, the baseline is a heavily blurred version of the same image: brightness and broad color distribution remain, while fine visual details disappear.

Intuition: Integrated Gradients measures which details become important as the image becomes sharp enough for the model to recognize the animal.

The Stress Test: Perturb the Background

AwA2 does not provide segmentation masks, so the current stress test uses an approximate mask: preserve a central region and perturb the outside. This is not a perfect animal mask; it is a controlled robustness test.

Perturbation What changes What it tests
gaussian_noise Adds random noise to approximate background pixels. Whether texture corruption outside the central region affects prediction.
color_shift Inverts RGB values in the approximate background. Whether background color statistics are influencing the model.
background_swap Replaces approximate background pixels with random noise. Whether removing the original context destabilizes the decision.

Let M in {0,1}^{H x W} be the approximate background mask, where M(i,j)=1 means that the pixel belongs to the region we perturb. The stressed image is constructed by preserving the animal region and replacing only the masked context. 

x_tilde = (1 - M) o x + M o T(x)

Gaussian noise:
T_noise(x) = clip(x + epsilon, 0, 1),      epsilon ~ Normal(0, sigma^2)

Color shift:
T_shift(x) = 1 - x

Background swap:
T_swap(x) = u,                             u ~ Uniform(0, 1)
The important control is that the target class for explanation can remain fixed to the original prediction. This lets us ask whether the explanation for the same class moves when only contextual pixels are modified.

Quantifying the Illusion

Visual inspection is necessary, but not sufficient. Phase 5 recomputes saliency maps after perturbation and measures how much they changed. The updated visual comparison also reports the animal predicted after each perturbation, so we can read the failure as a transition: original_prediction -> perturbed_prediction.

IoU top 20%

Converts each saliency map into a binary mask containing the top 20% most salient pixels. Low IoU means the explanation moved to a different region.

Spearman

Compares the rank ordering of all saliency pixels. Low correlation means the saliency structure changed broadly.

Confidence Delta

Measures how the model confidence changes after perturbation. This links explanation instability to prediction behavior.

Metric 1: IoU of the top-saliency support

A saliency map is continuous, so first we convert it into a binary support mask. Given a quantile threshold tau_q, the binary map keeps only the most salient pixels. In the experiments, q = 0.80, which means the top 20% of saliency values. 

B_q(S)(i, j) = 1[ S(i, j) >= tau_q(S) ]

IoU(S, S_tilde) =
| B_q(S) intersection B_q(S_tilde) | / | B_q(S) union B_q(S_tilde) |

If the prediction is stable but IoU is low, the model may still classify the image the same way while the explanation relocates to a different region. That is one concrete form of explanation instability.

Metric 2: Spearman rank correlation

IoU ignores the full ordering of saliency values. Spearman correlation compares the ranks of all pixels after flattening both maps. It is robust to monotonic rescaling, so it evaluates whether the spatial priority order is preserved rather than whether raw magnitudes match. 

r = rank(vec(S))
r_tilde = rank(vec(S_tilde))

rho_s(S, S_tilde) =
cov(r, r_tilde) / (std(r) * std(r_tilde))

Metric 3: Prediction and confidence stability

Explanation drift is most interesting when analyzed together with prediction behavior. The perturbation can preserve the predicted class, reduce confidence, or change the decision entirely. 

class_stable = 1[ y_hat(x) = y_hat(x_tilde) ]

Delta_conf = p(y_hat(x) | x) - p(y_hat(x) | x_tilde)

In the CSV this appears as original_prediction, perturbed_prediction, and prediction_changed. The notebook also builds a more readable prediction_transition field, for example antelope -> horse. This is important for the blog post because it turns an abstract instability metric into a concrete semantic confusion.

How to Read the Results Professionally

The analysis should not be summarized as "Grad-CAM is wrong" or "Integrated Gradients is wrong". A stronger claim is more precise: post-hoc saliency maps can be unstable under semantically controlled perturbations, especially when the classifier has learned contextual correlations.

Observation Mathematical signal Interpretation
Prediction remains unchanged, saliency changes strongly. class_stable = 1, low IoU, low rho_s. The decision is robust, but the explanation is not spatially stable.
Prediction confidence drops after background perturbation. Large positive Delta_conf. The model likely used contextual evidence as part of the decision.
The predicted animal changes after perturbing context. original_prediction -> perturbed_prediction, with prediction_changed = 1. The background intervention did not merely change confidence; it redirected the semantic class.
Heatmap moves from animal body to background. Top-saliency support shifts outside the preserved region. The explanation highlights predictive shortcuts rather than morphology.
Incorrect examples highlight plausible but wrong evidence. High confidence for y_hat != y with concentrated saliency. A confident heatmap can explain the wrong class, not the ground truth.
The professional conclusion is about reliability under intervention. A saliency method is more trustworthy when small, semantically irrelevant changes do not radically alter the explanation for the same prediction.

What This Teaches Us

The goal is not to claim that saliency maps are useless. They are useful exploratory tools. The problem is treating them as definitive explanations.

A heatmap can be visually convincing and still be unstable, context-sensitive, or disconnected from the causal features a human expects.

This is why the project naturally continues toward concept-based explainability. Instead of only asking where are the hot pixels?, we want to ask whether the model uses concepts like stripes, horns, fur, hooves, aquatic context, or body shape.

Stage Question Limitation
Saliency maps Where does the model appear to look? Pixel-level and post-hoc.
TCAV Which human concept influences the class score? Requires meaningful concept examples.
Concept Bottleneck Can the model predict concepts before the class? Requires concept supervision.
