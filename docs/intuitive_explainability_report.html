Saliency Maps Look Need Verification

Core claim

A heatmap is not an explanation by itself. It becomes useful only when it is compared across methods, tested under interventions, connected to concepts and checked against model behavior.

Overview Model Attribution Stress test Metrics Results Concepts Interpretation

The project explains a trained ResNet50 classifier on AwA2. The analysis starts from local saliency maps, then tests whether those maps remain meaningful when the input, target class and semantic interpretation are changed.

Training

Training minimizes cross-entropy between the true animal label and the predicted logits. The model is initialized from ImageNet weights. Early blocks are frozen, while layer3, layer4 and the final classifier are fine-tuned.

z = f_theta(x)
p(y=c | x) = exp(z_c) / sum_j exp(z_j)
predicted class = argmax_c p(y=c | x)
L(x,y) = -log p(y | x)

Overfitting controls

The baseline avoids overfitting through transfer learning, freezing early blocks, AdamW weight decay, validation-based learning-rate reduction and early stopping. Dropout is not added to the baseline ResNet50 head; dropout is used in the Concept Bottleneck head.

The checkpoint with the best validation behavior is used for all saliency, perturbation and concept analyses.

Attribution methods
Intervention

The background is changed to test shortcut behavior.

A saliency map can look plausible while the model depends on contextual evidence. The stress test keeps the animal region approximately stable and perturbs mostly background pixels.

Gaussian noise

Adds random noise to the background. If the prediction changes while the animal remains visible, the classifier is sensitive to non-semantic texture variation.

Color shift

Alters background color channels. This tests whether the model uses global color statistics as class evidence.

Background swap

Replaces background pixels with uniform noise. Large prediction or saliency drift indicates context dependence.

Interpretation. The goal is to measure whether the explanation follows the animal or moves toward unstable context when non-morphological evidence is perturbed.
Quantitative checks

Reliability is measured, not inferred from visual appeal.

How to read the numbers

The questions is if the same evidence remains important when the image is perturbed, and whether removing or inserting that evidence changes the model score in the expected direction.

Stability High IoU and high Spearman mean the saliency structure survives the intervention.
Faithfulness Good deletion and insertion curves mean salient pixels are behaviorally important.
Context use High background ratio means the explanation is using pixels outside the animal proxy.

Spatial stability: Top-k IoU

Metric 1 of 6

Spatial stability Top-k IoU

Binarize the top 20% most salient pixels in the original and perturbed images, then compare their overlap.

IoU = |S_original intersect S_perturbed| / |S_original union S_perturbed|
High is good for stability. Low means the explanation moved.

Use it when: you want to know whether the highlighted region remains spatially consistent.

Rank stability Spearman

Flatten both saliency maps and compare the rank ordering of pixels, not their absolute values.

rho = corr(rank(M_original), rank(M_perturbed))
High is good. Low means the saliency priority changed.

Local attribution comparison

Grad-CAM and Score-CAM produce more object-concentrated maps on the audited examples, while Integrated Gradients and Expected Gradients distribute substantially more attribution outside the animal mask. This does not make CAM methods automatically faithful; it only says their spatial support is more aligned with the proxy object region in this run.

Stress-test behavior

Background interventions are not visually aimed at the animal morphology, but they still change both predictions and saliency maps. This is the central diagnostic of the project: an explanation that moves when only context is perturbed is not sufficient evidence of morphology-based reasoning.

Faithfulness and sanity checks

Deletion/insertion results test whether highlighted pixels affect the model score. In the advanced attribution audit, Grad-CAM and Score-CAM have larger insertion AUC than the gradient-path methods, while Expected Gradients has the smallest faithfulness gap on the inspected subset.

Concept-level evidence

TCAV and the bottleneck model move the project beyond pixel heatmaps. TCAV asks whether a learned concept direction increases a class logit; the bottleneck model forces prediction through explicit concept estimates before the final class decision.

A high TCAV score means sensitivity along a concept direction, not proof that the visible object literally contains that concept. Concept methods still need sanity checks.
Semantic level

Concepts make the explanation closer to human reasoning.

Pixel heatmaps describe spatial sensitivity. Concept-level methods ask whether the model is sensitive to human-readable animal attributes.

TCAV

TCAV learns a direction in activation space for a concept and checks whether moving along that direction increases the class score. It provides a class-level concept sensitivity rather than a pixel map.

TCAV(c, concept) = fraction of examples where dS_c / dv_concept > 0

Concept Bottleneck Model

A bottleneck model predicts concepts before predicting the class. This makes the semantic interface explicit: errors can be inspected at the concept level and then at the class level.

x -> predicted concepts -> predicted class

Final position

Grad-CAM, Score-CAM, Integrated Gradients and Expected Gradients are useful diagnostic tools, but they should not be presented as standalone proof. A reliable explainability analysis combines several independent signals before claiming that a heatmap reflects the evidence actually used by the model.
