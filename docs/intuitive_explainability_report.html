Explainability

Principle 1 of 7

Core definition Explainability

Explainability is the study of how to make a model's behavior inspectable. In this project it means asking which visual evidence supports an animal-class prediction and whether that evidence is stable under controlled tests.

image x -> model f(x) -> logits S -> explanation E(x, class)
  • Prediction: the class assigned by the model.
  • Explanation: an auxiliary object that tries to describe why that score was produced.
  • Audit: the set of tests used to check whether the explanation is reliable.
Local attribution Saliency map

A local saliency map assigns an importance value to pixels or image regions for one image and one target class. It is local because it explains a single prediction, not the global behavior of the classifier. When this numeric map is displayed with colors, it becomes the heatmap shown on top of the original image.

M_c(x)[i, j] = importance of pixel or region (i, j) for class c
  • High value: the method considers that region important for the target score.
  • Low value: the method assigns little contribution to that region.
  • Heatmap: the colored visualization of the saliency map, usually overlaid on the original image.
  • Caveat: a highlighted region is a hypothesis about evidence, not proof of causal use.
Classifier output Logits and softmax

ResNet50 first produces logits: unnormalized class scores. Softmax converts logits into probabilities by comparing each score with all other class scores.

p_c = exp(S_c) / sum_j exp(S_j)
  • Logit S_c: the raw target score usually explained by attribution methods.
  • Softmax p_c: the normalized confidence shown to the user.
  • Important distinction: explanations often target logits because softmax couples all classes together.
Differential sensitivity Gradient attribution

Gradient-based explanations ask how the target score would change for an infinitesimal change in the input pixels. They are sensitive to local behavior around the current image.

G_c(x) = dS_c(x) / dx
  • Large gradient: a small pixel change can strongly affect the target logit.
  • Useful for: detecting fine local sensitivity and edge/texture dependence.
  • Limitation: high sensitivity is not always the same as semantic importance.
Activation localization CAM principle

CAM-style methods use convolutional feature maps to localize class-discriminative regions. In this project Grad-CAM and Score-CAM operate near the last semantic layer of ResNet50.

M_c = ReLU(sum_k alpha_k^c A_k)
  • A_k: feature map from a convolutional layer.
  • alpha_k: importance weight for that feature map.
  • Consequence: maps are class-discriminative but spatially coarse before upsampling.
Reference dependence Baseline-path attribution

Integrated Gradients and Expected Gradients compare the image with one or more reference baselines. Importance is accumulated along the path from baseline to image.

IG_i(x) = (x_i - x'_i) integral_0^1 dS_c(x' + a(x - x')) / dx_i da
  • Baseline: the reference image used to define absence of evidence.
  • Blurred baseline: used here to preserve average illumination better than a black image.
  • Expected Gradients: averages this idea across multiple baselines.
Reliability criteria Faithfulness and stability

A saliency map should not only look plausible. It should identify evidence that affects the model score and should remain reasonably stable under interventions that do not change the relevant object evidence.

reliable explanation = visual plausibility + behavioral tests + stability checks
  • Faithfulness: removing important pixels should reduce the target score.
  • Stability: saliency should not drift arbitrarily after controlled changes.
  • Concept alignment: semantic evidence should agree with human-readable animal attributes when possible.

Skip connection: residual learning output = F(x) + x The block learns a correction to the input instead of a fully new representation. This helps deep networks train reliably.
Downsampling: from pixels to concepts 56 -> 28 -> 14 -> 7 Layer2, layer3 and layer4 reduce spatial size with stride 2. When channels or size change, the skip path uses a 1 x 1 projection.

Attribution methods

Each attribution method answers a different question.

The same ResNet50 prediction is inspected with activation-based, gradient-path and perturbation-based methods. The notebook results below report behavior on the evaluated examples, not universal method rankings.

Grad-CAM

Method 1 of 6

Grad-CAM

Grad-CAM explains a target logit using the last convolutional feature maps. It computes gradient weights for each channel in layer4, combines the weighted activation maps, applies ReLU and upsamples the result to image size.

alpha_k^c = mean_{i,j}(dS_c / dA^k_{i,j})
M_c = ReLU(sum_k alpha_k^c A^k)

It is good for broad localization, but the native map is coarse because layer4 is only about 7 x 7 before upsampling.

Score-CAM

Score-CAM uses activation maps without gradients. Each activation map is upsampled and used as a mask on the input image; the masked image is forwarded through the model and the target score becomes the channel weight.

w_k^c = softmax(S_c(x * upsample(A_k)))
M_c = ReLU(sum_k w_k^c A_k)

This makes it a useful comparison to Grad-CAM: if both agree, the last convolutional features are carrying a consistent localization signal.

Integrated Gradients

Integrated Gradients attributes a prediction by accumulating gradients along a path from a baseline to the real image. In this project the baseline is a strongly blurred version of the image, not a black image, to preserve average brightness.

IG_i(x) = (x_i - x'_i) integral_0^1 dS_c(x' + a(x-x')) / dx_i da

It provides fine-grained pixel attribution, but it is sensitive to baseline choice and can distribute importance over texture and background.

Expected Gradients

Expected Gradients generalizes Integrated Gradients by sampling multiple baselines and interpolation points. The attribution is an expectation over reference images and path positions.

EG_i(x) = E_{x',a}[(x_i - x'_i) dS_c(x' + a(x-x')) / dx_i]

It reduces dependence on a single baseline, but in this project it still assigns substantial attribution outside the animal proxy mask.

SmoothGrad

SmoothGrad averages gradients over noisy copies of the same image. The objective is to reduce high-frequency gradient noise and reveal more persistent local sensitivity patterns.

M_smooth(x) = (1/N) sum_n M_grad(x + epsilon_n)

It is a denoising wrapper around gradients. It does not turn gradient sensitivity into causal evidence, so it should be compared with perturbation methods.

Occlusion

Occlusion removes or replaces image patches and measures how much the target score changes. It is slower than gradients, but closer to an intervention because the input is actually modified.

Importance(patch) = S_c(x) - S_c(x with patch removed)

It is useful as a sanity check: if a saliency map highlights evidence that occlusion says is behaviorally irrelevant, the visual explanation should be treated cautiously.

Intervention

The background is changed to test shortcut behavior.

A saliency map can look plausible while the model depends on contextual evidence. The stress test keeps the animal region approximately stable and perturbs mostly background pixels.

Gaussian noise

Adds random noise to the background. If the prediction changes while the animal remains visible, the classifier is sensitive to non-semantic texture variation.

Color shift

Alters background color channels. This tests whether the model uses global color statistics as class evidence.

Background swap

Replaces background pixels with uniform noise. Large prediction or saliency drift indicates context dependence.

Interpretation. The goal is to measure whether the explanation follows the animal or moves toward unstable context when non-morphological evidence is perturbed.

Spatial stability Top-k IoU

Binarize the top 20% most salient pixels in the original and perturbed images, then compare their overlap.

What to observe

This card is the first sanity check: the classifier predicts the right class, then different attribution methods are computed for that predicted class.

  • Input gradients show local pixel sensitivity and often emphasize edges, texture changes and high-frequency details.
  • Grad-CAM uses the last convolutional block, so the heatmap is spatially coarse and class-discriminative rather than pixel-precise.
  • Integrated Gradients attributes the prediction along a path from a blurred baseline to the real image; bright regions indicate pixels that accumulate evidence for the target logit.

How to read errors

These maps should be read with the predicted label, not only with the ground truth. The explanation answers: which evidence supported the wrong class score?

  • If the heatmap covers the animal, the model may be using morphology but confusing similar visual patterns.
  • If the heatmap moves to the background, the error is compatible with contextual shortcut behavior.
  • If methods disagree, the failure is not explained by one universal notion of importance; gradient, activation and path methods are measuring different signals.

What Grad-CAM sees

Grad-CAM weights the layer4 feature maps by the gradient of the target logit. It therefore highlights broad semantic regions whose activations increase the class score.

  • Coarse localization is expected because the native feature map is around 7 x 7 before upsampling.
  • Stable animal focus supports visual plausibility, but does not prove causality.
  • Map drift after background swap means that the target score or the CAM weights changed when non-animal pixels were modified.

What Score-CAM adds

Score-CAM does not use gradients. It masks the input with activation maps and measures how much each masked image increases the target score.

  • Gradient-free weighting makes it a useful comparison against Grad-CAM.
  • Similar maps to Grad-CAM suggest that the last convolutional activations carry a consistent localization signal.
  • Differences under perturbation reveal whether gradient weights or score responses are driving the explanation.

What Integrated Gradients sees

Integrated Gradients accumulates input gradients along the interpolation path from the baseline to the real image. In this project the baseline is blurred, so mean brightness and global color are less artificially destroyed than with a black image.

  • Fine-grained attribution can spread across texture, contours and background pixels.
  • Baseline sensitivity is part of the method, not an implementation bug.
  • Low overlap after perturbation indicates that the pixel ranking changes substantially when context is modified.

What Expected Gradients changes

Expected Gradients generalizes Integrated Gradients by averaging over multiple baselines. The objective is to reduce dependence on one arbitrary reference image.

  • Distributed attribution is expected because evidence is averaged over several paths.
  • Background mass is important here: the notebook summary shows a high background saliency ratio for Expected Gradients.
  • Interpretation should focus on stability and faithfulness metrics, not only on whether the map looks smooth.

Why this matters

This diagnostic checks whether different explanation families agree on the same evidence. A visually convincing heatmap is weaker if it does not match occlusion or if it remains similar to randomized controls.

  • SmoothGrad averages noisy gradients to reduce visual noise.
  • Occlusion is closer to an intervention because image patches are removed and the score response is measured.
  • Randomized checks help detect explanations that look structured even when they should not be trusted.

What TCAV sees

TCAV does not produce a pixel map. It learns a direction in activation space for a human concept and checks whether moving along that direction increases a target class logit.

  • High score means the class score is often positively sensitive to that concept direction.
  • Example result: stripes -> tiger is high, which is semantically plausible.
  • Important caveat: TCAV reports representation sensitivity, not visual proof that the concept is present in a specific image.

What the bottleneck changes

The Concept Bottleneck Model forces an explicit intermediate representation: image to concepts, then concepts to class. This makes errors inspectable at a semantic level.

  • Concept accuracy tests whether attributes such as stripes, flippers or horns are predicted reliably.
  • Class accuracy tests whether those predicted concepts are sufficient for the animal classifier.
  • Interventions can change a concept value and observe how the class probability changes, making the explanation more operational than a heatmap.
Semantic level

Concepts make the explanation closer to human reasoning.

Pixel heatmaps describe spatial sensitivity. Concept-level methods ask whether the model is sensitive to human-readable animal attributes.

How concept information is built in this project

AwA2 provides class-level semantic attributes such as stripes, horns, hooves, flippers, furry and ocean. These attributes are not pixel masks. They are human-readable concept annotations attached to animal classes. The project uses them to move from local pixel attribution to semantic explanation: instead of asking only where the model looked, we ask whether the internal representation is sensitive to concepts that a human would use to describe the animal.

1

Choose a concept

Select an AwA2 attribute, for example stripes, horns or flippers, and identify classes where that concept is present or absent.

2

Extract activations

Pass images through ResNet50 and collect intermediate activations, typically after a semantic convolutional block such as layer3.

3

Learn concept evidence

Train or evaluate a simple concept model in activation space. The concept becomes a direction or an intermediate variable.

4

Test class influence

Measure whether changing concept evidence changes the target class score, probability or predicted class.

TCAV intuition

TCAV card 1 of 2

TCAV as a full concept-level flashcard

TCAV starts from a human concept such as stripes or flippers. It collects examples where that concept is present and absent, extracts internal activations from ResNet50, and learns a direction in activation space that separates the two groups.

  • What the diagram shows: concept examples become activation vectors; a linear separator defines the Concept Activation Vector.
  • What the algorithm measures: whether the target class logit increases when the activation moves along that concept direction.
  • How to interpret it: a high score means concept sensitivity at class level, not a pixel localization map.

TCAV: Testing with Concept Activation Vectors

TCAV treats a concept as a direction in the model's activation space. For a chosen layer, images that strongly contain a concept are used as positive examples, while images that do not contain it are used as negative examples. A linear separator is trained between the two groups. The normal vector of this separator is the Concept Activation Vector.

How the score is computed

After the CAV is learned, TCAV evaluates target-class images and counts how often the class logit increases in the concept direction.

  • Input: images, class labels, AwA2 concept annotations and a selected ResNet layer.
  • Mechanism: learn a linear concept direction in activation space.
  • Score: count positive directional derivatives.
  • Interpretation: concept evidence tends to increase the class score.
z_l(x) = activation of image x at layer l
v_k = CAV direction for concept k
sensitivity(x, c, k) = gradient_z S_c(z_l(x)) dot v_k
TCAV(c, k) = (1 / N) sum_i 1[gradient_z S_c(z_l(x_i)) dot v_k > 0]
In the notebook results, concept directions such as stripes -> tiger and flippers -> dolphin are strong examples of semantic sensitivity. This is class-level evidence, not a pixel-level localization map.

Concept Bottleneck as a full concept-level flashcard

A Concept Bottleneck Model changes the model architecture: the image is first mapped to a vector of concepts, and the final animal prediction is computed from those concepts rather than directly from opaque image features.

  • What the diagram shows: image features are compressed into concepts such as horns, hooves, stripes and flippers before class probabilities are produced.

Concept Bottleneck Model

A Concept Bottleneck Model changes the prediction path. Instead of predicting the animal class directly from image features, the model first predicts a vector of human-readable concepts and then predicts the class from those concepts. This creates an explicit semantic interface that can be inspected and intervened on.

How the bottleneck is trained and read

The model has two coupled objectives: predicting the correct animal class and predicting the semantic attributes. This creates a measurable intermediate layer.

  • Concept head: predicts attributes such as horns, stripes, flippers or furry.
  • Class head: receives the concept vector and assigns class logits.
  • Training objective: combines class loss and concept prediction loss.
  • Interpretability gain: errors can be decomposed at concept level.
image x -> feature encoder h(x) -> predicted concepts c_hat -> logits -> softmax probabilities
L = CE(y, y_hat) + lambda * L_concept(c, c_hat)
p(y | x) = softmax(g(c_hat))
In the project output, the bottleneck classifier reaches 0.805 test class accuracy and 0.754 concept binary accuracy on the evaluated subset. This makes the model less opaque than a pure end-to-end classifier, but it also exposes concept prediction errors.

Concept interventions

Concept interventions make the explanation operational. After the bottleneck predicts a concept vector, one concept can be forced to a chosen value and the class probability can be recomputed. This answers a causal-style question: if the model believed this animal had more stripes, horns or flippers, how would the class probability change?

intervention_delta = p(y = c | c_hat_k = 1) - p(y = c | c_hat_k = 0)

What this reveals

If changing one concept strongly changes a class probability, that class decision is semantically sensitive to the concept. This is more directly testable than simply looking at a heatmap.

What it does not prove

An intervention on a concept vector does not prove that the original image visibly contains the concept. It proves that the downstream classifier uses that concept variable when it is available.

Why concept methods complement saliency maps

Saliency maps are local and spatial: they highlight pixels or regions for one image and one target class. TCAV and bottleneck models are semantic: they evaluate whether the model's representation and prediction pathway are aligned with concepts that humans can name. The two families answer different questions and should be interpreted together.

  • Grad-CAM, Score-CAM, IG and Expected Gradients: where is the image evidence located for this prediction?
  • TCAV: is a class score sensitive to a semantic concept direction in activation space?
  • Concept Bottleneck: can the prediction be routed through explicit concepts, and can those concepts be intervened on?
  • Project interpretation: a reliable explanation should connect local attribution, stress-test stability and concept-level behavior.
Final conclusions

Explainability is reliable only when visual evidence, interventions and concepts agree.

The project does not treat a heatmap as an explanation by itself. A saliency map is interpreted as a hypothesis about the evidence used by the model, and that hypothesis is accepted only when it survives quantitative and semantic checks.

Observed result Background interventions change both predictions and saliency maps. For example, background-swap IoU is 0.517 for Grad-CAM and 0.278 for Integrated Gradients.
Conclusion A visually plausible heatmap can still be context-sensitive. The explanation must therefore be evaluated against controlled perturbations, not only by visual inspection.
Faithfulness metrics IoU, Spearman correlation, deletion AUC and insertion AUC measure whether salient regions are stable and behaviorally relevant.
Observed result Score-CAM and Grad-CAM show stronger localization and insertion behavior than the more distributed gradient-path methods on the inspected subset.
Conclusion The most useful explanation is not necessarily the sharpest heatmap. It is the explanation whose highlighted evidence changes the model score in the expected direction.
Sanity checks SmoothGrad, Occlusion and randomized controls test whether gradients, smoothing and intervention-based evidence tell the same story.
Observed result Vanilla gradients agree weakly with SmoothGrad and Occlusion: the reported IoUs are 0.253 and 0.201 respectively.
Conclusion Gradient sensitivity alone is not enough. A reliable audit must compare saliency with perturbation-based evidence and randomized references.
Concept-level evidence TCAV and the Concept Bottleneck Model move the analysis from pixels toward human-readable animal attributes.
Observed result The bottleneck classifier reaches 0.805 test class accuracy and 0.754 concept binary accuracy on the evaluated subset.
Conclusion Concepts make the explanation more inspectable, but they also expose a second source of error: the model can be wrong because the class classifier is wrong or because the concept predictor is wrong.

Final conclusion

The project supports a conservative interpretation of explainability in computer vision: attribution maps are useful diagnostic instruments, not standalone proof of reasoning. The ResNet50 classifier can assign high confidence to an animal class, and a saliency method can produce a visually convincing heatmap, but this does not guarantee that the model relied on animal morphology. The audit becomes meaningful only when local attribution, background perturbations, faithfulness metrics, sanity checks and concept-level explanations are read together.

The strongest conclusion is therefore methodological: explainability should be treated as an evidence pipeline. A claim is credible when the highlighted pixels remain stable under irrelevant changes, when removing those pixels changes the score, when different attribution families give compatible signals, and when semantic concepts support the same interpretation.
